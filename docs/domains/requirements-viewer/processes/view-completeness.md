# View Completeness

Understand the implementation status of requirements across the hierarchy.

## Trigger

User wants to see how much of the system is complete.

## Steps

1. User selects a completeness view
2. System displays the requirements hierarchy with completion status highlighted
3. User sees which requirements are done vs pending at each level (Domain, Subdomain, Capability)

## Outcome

User understands the overall progress and can identify areas that need attention.
